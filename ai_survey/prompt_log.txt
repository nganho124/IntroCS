Ask: I want to create a tracker.py where I can use command line such as "python tracker.py" to:
* add new tast as: add "new task"
* print list of tasks in tracker.py as: list
* mark a task a finish as: finish "task id"
* delete a task with id: delete "task id"
To do that, I also want to have the file named tasks.txt where tasks are stored like: task_id[TAB]finish_state[TAB]task_name[NEWLINE]

Also, all the command when I run, either success or not will be stored in a file called: audit.txt

Can you help me to design these 3 files?


Answer:
 I've designed and implemented all three files for you. Here's what I created:
📁 Files Created
1. tracker.py - The Main Program

Uses object-oriented design with a TaskTracker class
Handles all four commands: add, list, finish, delete
Includes comprehensive error handling
Automatically creates and manages tasks.txt and audit.txt
Uses only Python standard library (no pip install needed)

2. tasks.txt - Task Storage (Auto-generated)
Format: task_id[TAB]finish_state[TAB]task_name[NEWLINE]
Example content:
1	1	Buy milk
3	0	Call mom
3. audit.txt - Audit Log (Auto-generated)
Format: timestamp | command | status
Example content:
2026-01-20 17:12:08 | add Buy milk | S
2026-01-20 17:12:41 | delete 999 | F
🎯 Key Design Decisions

ID Generation: Sequential IDs starting from 1, never reused
Finish State: 0 = unfinished, 1 = finished
Audit Format: Pipe-separated for clarity (timestamp | command | S/F)
Error Handling: All edge cases handled (invalid IDs, non-existent tasks, etc.)
File Safety: tasks.txt is rewritten completely; audit.txt is append-only

🧪 How to Use
